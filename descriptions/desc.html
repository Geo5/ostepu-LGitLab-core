Dieser kann entweder über `Git` selbst oder die Weboberfläche von `GitLab` erzeugt werden. *Achtung: Wenn Sie eine Aufgabe erneut einsenden möchten, dann müssen Sie den bisherigen `TAG` vorher löschen (`GitLab` wird Ihnen ansonsten eine Fehlermeldung ausgeben).* ### Einsenden 1. Sie müssen in Ihrem Repository den Commit auswählen, welchen Sie einsenden möchten. (als Einsendezeitpunkt wird der Zeitstempel des Commit verwendet) ![](I.png) 2. Jetzt können Sie für diesen Commit einen `TAG` anlegen. ![](G.png) 3. Der Name des TAG muss die Form `SUBMIT__` haben. Dabei darf der `Serienname` keine Leerzeichen enthalten (aus *Serie 4* wird daher *Serie4*) ![](H.png) Beispiele: * `SUBMIT_SERIE4_1} (*Serie 4*, Aufgabe 1)` * `SUBMIT_serie4_2b} (*Serie 4*, Aufgabe 2b)` * `SUBMIT_BONUSSERIE_5A} (die Serie mit dem Namen *Bonusserie*, Aufgabe 5a)` 4. Der TAG kann nun erzeugt werden. ![](J.png) ### Meldungen Sie erhalten eventuelle Fehlermeldungen von der Übungsplattform in der Weboberfläche von GitLab. Sie finden diese Meldungen in den Beschreibungstexten der Tags.
